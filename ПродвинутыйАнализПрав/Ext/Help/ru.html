Форма обработки позволяет гибко настраивать параметры для анализа прав доступа. Вы можете использовать различные поля для фильтрации результатов.

### Элементы управления на форме:

*   **Объект метаданных** Введите полное наименование объекта метаданных (например, "Документ.РеализацияТоваровУслуг", "Справочник.Номенклатура"). Обработка будет искать права, связанные с этим объектом.

*   **Роли:** Это поле позволяет выбрать роль, для которой вы хотите проанализировать права. Если вы хотите проанализировать права для Профилей, выберите "Показывать профили".

*   **Наименование типа доступа:** Введите наименование типа доступа (например, "Чтение", "Изменение", "Добавление", "Просмотр", "Редактирование"). Это позволит отфильтровать права по конкретным действиям.

*   **Профиль:** Если в поле "Роли" выбрано значение "Профиль", то это поле становится доступным для выбора конкретного профиля доступа.


### Фильтрация и заполнение таблицы:

Все указанные выше поля работают как фильтры. Вы можете использовать их по отдельности или комбинировать для получения более точных результатов. Например, вы можете найти: 

*   "Полные права" для объекта "Документ"; 
*   Права "Чтение" для роли "Администратор" по объекту "Справочник.Пользователи" и т.д.

Важно: если введенные фильтры взаимоисклучают друг друга (например, вы ищете конкретную роль и профиль, которые не могут существовать одновременно), результат будет пустым.

После того как вы установили необходимые фильтры, нажмите кнопку **"Заполнить таблицу"**.

### Результаты анализа:

После нажатия кнопки "Заполнить таблицу" обработка выполнит анализ и отобразит результаты в табличной части формы. В нижней части формы, в поле **"Количество объектов"**, будет показано общее количество найденных объектов, соответствующих вашим критериям фильтрации.
